USE CASES & TESTING

Use cases define how users interact with the system. Test cases go to QA; UAT cases go to the business team.

BA Tip: Write use cases from the user's perspective — "what does the user need to accomplish?" Link each use case to at least one functional requirement.
For the QA/Testing Team: Test cases are technical and system-focused. Write steps clearly enough that a tester can follow them without asking questions.
For the Business Team: UAT cases are written in plain business language using Given/When/Then format. The business team uses these to confirm the solution meets their needs.
